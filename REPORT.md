## 🎯 Uppgiftens Krav:
# Inlämningsuppgift Todo

I denna inlämningsuppgift kommer ni att skapa er egen todo-lista i en react-applikation.
Sidan skall visa ett antal punkter som skall göras. Dessa skall då komma upp på skärmen i form av en lista. När uppgiften är slutförd skall användaren kunna markera uppgiften som slutförd och uppgiften skall då tas bort från listan.

## Betyg G

- Skapa en hårdkodad lista med punkter att göra (hitta på egna punkter, dessa skall inte bara vara en text)
- Presentera listan på skärmen, helst med lite kontroll. Detta betyder i en html-struktur t.ex. i en ul/li-lista
- Implementera klickhändelse för att hantera borttagandet av en todo.
- Todo markeras som klar/tas bort från skärmen och markeras som klar i javascript-listan.

## Betyg VG

- Alla punkter under G
- Kunna visa även klara händelser och klicka tillbaka den så att de blir oklara igen.
- Skapa ett formulär som tillåter att en användare skapar nya todos efterhand.
- Använda lifting state up för att dela upp dina komponenter bättre.
- Kunna sortera ordningen på dina todos.
- Implementera ett valfritt grafiskt ramverk till din todolista, t.ex. material ui eller tailwind.
- Egen css får gärna skrivas och då skall ni ha en bra struktur och använda flex eller grid på ett bra sätt.

## 🔍 ESLint-varningar:


## 🏆 **Betyg: VG**
📌 **Motivering:** Koden uppfyller alla krav för VG. Projektet använder en hårdkodad lista, visar todo-listan på skärmen i en organiserad struktur, och stöder klickhändelser för att hantera borttagandet av todos. Slutförda uppgifter kan markeras och visas, och användaren kan skapa nya todos via ett formulär. Projektet visar användning av lifting state up och erbjuder sortering av todos. Tailwind CSS används som ett grafiskt ramverk och CSS-strukturen är tydlig med användning av flexbox och grid.

💡 **Förbättringsförslag:**  
Koden kan förbättras genom att ytterligare modularisera komponenter, och se till att komponenterna har mer specifika ansvarsområden. Det kan vara lämpligt att bryta ner större komponenter som TodosHandle till flera mindre komponenter för bättre underhållbarhet. Dessutom bör projektet överväga mer enhetstester för att förbättra kodens tillförlitlighet.
